Get started with fzf and learn the basics in just a few minutes.

<Steps>
  <Step title="Verify installation">
    After installing fzf, verify it's working by running:

    ```bash
    fzf --version
    ```

    You should see the version number (0.68.0 or later).
  </Step>

  <Step title="Try your first search">
    Let's start with the simplest example. Run fzf without any arguments:

    ```bash
    fzf
    ```

    fzf will traverse the current directory and display all files. Try typing to search!

    **Navigation:**
    - Type to fuzzy search
    - `CTRL-K` / `CTRL-J` or arrow keys to move
    - `Enter` to select
    - `ESC` or `CTRL-C` to exit
  </Step>

  <Step title="Use with pipes">
    fzf reads from STDIN and writes to STDOUT, making it perfect for Unix pipes:

    ```bash
    # Find and open a file in vim
    find * -type f | fzf | xargs vim

    # Or use command substitution
    vim $(find * -type f | fzf)
    ```
  </Step>

  <Step title="Set up shell integration">
    Shell integration adds powerful key bindings. Add this to your shell config:

    <Tabs>
      <Tab title="Bash">
        Add to `~/.bashrc`:
        ```bash
        # Set up fzf key bindings and fuzzy completion
        eval "$(fzf --bash)"
        ```
      </Tab>
      <Tab title="Zsh">
        Add to `~/.zshrc`:
        ```bash
        # Set up fzf key bindings and fuzzy completion
        source <(fzf --zsh)
        ```
      </Tab>
      <Tab title="Fish">
        Add to `~/.config/fish/config.fish`:
        ```fish
        # Set up fzf key bindings
        fzf --fish | source
        ```
      </Tab>
    </Tabs>

    Reload your shell:
    ```bash
    source ~/.bashrc  # or ~/.zshrc for zsh
    ```
  </Step>

  <Step title="Try the key bindings">
    After shell integration, you get these powerful key bindings:

    **CTRL-T** - Find files and directories
    ```bash
    # Press CTRL-T at any time to search for files
    vim <CTRL-T>  # Select a file to open
    ```

    **CTRL-R** - Search command history
    ```bash
    # Press CTRL-R to search your shell history
    <CTRL-R>  # Type to search, Enter to select
    ```

    **ALT-C** - Change directory
    ```bash
    # Press ALT-C to search for a directory and cd into it
    <ALT-C>  # Select a directory
    ```
  </Step>
</Steps>

## Common Use Cases

### File Navigation

Search and open files quickly:

```bash
# Find and edit a file
vim $(fzf)

# Better: Use become() to handle CTRL-C and empty results
fzf --bind 'enter:become(vim {})'
```

### Process Management

Find and kill processes:

```bash
# Interactive process killer
ps -ef | fzf | awk '{print $2}' | xargs kill -9
```

### Git Integration

Work with Git more efficiently:

```bash
# Checkout a branch
git branch | fzf | xargs git checkout

# View a commit
git log --oneline | fzf | awk '{print $1}' | xargs git show
```

## Search Syntax

fzf supports powerful search syntax:

| Pattern   | Match Type              | Example                          |
|-----------|-------------------------|----------------------------------|
| `sbtrkt`  | Fuzzy match             | Matches "subprocess.txt"         |
| `'wild`   | Exact match             | Matches "wild" exactly           |
| `^music`  | Prefix match            | Matches "music.mp3"              |
| `.mp3$`   | Suffix match            | Matches "song.mp3"               |
| `!fire`   | Inverse match           | Excludes items with "fire"       |
| `go$ \| rb$ \| py$` | OR operator  | Matches files ending in go, rb, or py |

<Note>
  Try typing `^src .go$` to find all Go files in the src directory!
</Note>

## Display Options

### Inline Mode

Use `--height` to show fzf below the cursor:

```bash
fzf --height 40% --layout reverse --border
```

### Tmux Popup

If you're using tmux, fzf can open in a popup:

```bash
fzf --tmux center
fzf --tmux bottom,40%
```

<Tip>
  Add these options to `FZF_DEFAULT_OPTS` to make them permanent:
  ```bash
  export FZF_DEFAULT_OPTS='--height 40% --layout reverse --border'
  ```
</Tip>

## Environment Variables

Configure fzf's behavior with environment variables:

<ParamField path="FZF_DEFAULT_COMMAND" type="string">
  Command to use for generating the file list when fzf runs without STDIN input.

  ```bash
  # Use fd instead of find (respects .gitignore)
  export FZF_DEFAULT_COMMAND='fd --type f --strip-cwd-prefix'
  ```
</ParamField>

<ParamField path="FZF_DEFAULT_OPTS" type="string">
  Default options for fzf.

  ```bash
  export FZF_DEFAULT_OPTS='--height 40% --layout reverse --border --inline-info'
  ```
</ParamField>

<ParamField path="FZF_CTRL_T_COMMAND" type="string">
  Command for CTRL-T key binding.

  ```bash
  export FZF_CTRL_T_COMMAND="$FZF_DEFAULT_COMMAND"
  ```
</ParamField>

## Preview Window

Add a preview window to see file contents:

```bash
# Preview with cat
fzf --preview 'cat {}'

# Preview with syntax highlighting (requires bat)
fzf --preview 'bat --color=always {}'
```

Make it permanent in `FZF_CTRL_T_OPTS`:

```bash
export FZF_CTRL_T_OPTS="
  --preview 'bat -n --color=always {}'
  --bind 'ctrl-/:change-preview-window(down|hidden|)'"
```

<Warning>
  Don't add `--preview` to `FZF_DEFAULT_OPTS` as it won't work with non-file inputs like process lists or command history.
</Warning>

## Complete Example

Here's a complete setup combining multiple features:

```bash
# ~/.bashrc or ~/.zshrc

# Use fd for file listing (respects .gitignore)
export FZF_DEFAULT_COMMAND='fd --type f --strip-cwd-prefix --hidden --follow --exclude .git'

# Default options
export FZF_DEFAULT_OPTS='
  --height 40%
  --layout reverse
  --border
  --inline-info
  --color=fg:#c0caf5,bg:#1a1b26,hl:#bb9af7
  --color=fg+:#c0caf5,bg+:#292e42,hl+:#7dcfff
  --color=info:#7aa2f7,prompt:#7dcfff,pointer:#7dcfff
  --color=marker:#9ece6a,spinner:#9ece6a,header:#9ece6a'

# CTRL-T options
export FZF_CTRL_T_COMMAND="$FZF_DEFAULT_COMMAND"
export FZF_CTRL_T_OPTS="
  --preview 'bat -n --color=always {}'
  --bind 'ctrl-/:change-preview-window(down|hidden|)'"

# ALT-C options
export FZF_ALT_C_OPTS="
  --preview 'tree -C {} | head -200'"

# Set up fzf key bindings and fuzzy completion
eval "$(fzf --bash)"  # or source <(fzf --zsh) for zsh
```
